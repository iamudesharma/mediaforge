# image_forge

[![pub package](https://img.shields.io/pub/v/image_forge.svg)](https://pub.dev/packages/image_forge)
[![License](https://img.shields.io/badge/License-Apache%202.0-blue.svg)](LICENSE)
[![Platform](https://img.shields.io/badge/Platform-Android%20%7C%20iOS%20%7C%20macOS%20%7C%20Windows%20%7C%20Linux-green.svg)](#platform-matrix)

High-performance Rust image processing engine for Flutter, powered by **flutter_rust_bridge** (FRB) v2. This package contains the core headless engine with optimized SIMD CPU algorithms, `wgpu` GPU compute pipelines (WGSL shaders), face landmarks masking/beauty warps, and vector/raster layer composition.

> [!NOTE]
> This is a **headless engine** containing Rust core APIs and FFI bindings. It does **not** include Flutter UI widgets. For the complete ready-to-use editor UI, check out [`image_forge_editor`](../image_forge_editor/). For the GPU texture rendering bridge, see [`pixel_surface`](../pixel_surface/).

## Key Subsystems

- **Core Image Operations**: Ultra-fast resizing, cropping, rotating, and format transcode (JPEG, PNG, WebP, AVIF).
- **Advanced Buffer Manipulation**: Zero-copy pixel array FFI transfers and progressive image decoding.
- **GPU Shader Pipeline**: Separation of preview/export filters running on `wgpu` using WGSL compute shaders.
- **Face Analysis & Beauty**: Real-time regional beauty, skin smoothing, eye brightening, lip tinting, and facial morphing/warps.
- **Paint & Layer Composition**: baking multi-layer sticker overlays, text boxes, freehand vector paint strokes (marker, highlighter, neon), and erasers.
- **Memory Pooling**: Zero-allocation buffer reuse system to prevent heap allocations on intensive render ticks.
- **Temporal Stream Filtering**: Live camera face landmark stabilizer using Exponential Moving Average (EMA).

## API Reference

`image_forge` exposes clean synchronous APIs generated by `flutter_rust_bridge`. 

### 1. Core Image Processing (`api/image.rs`)

*   `init_app()`: Configures the CPU thread pool size (`rayon`) and initializes user utility hooks.
*   `resize_image(bytes, width, height, algorithm, format, quality, fixExif, backend)`: Resizes image files using Nearest, Box, Hamming, CatmullRom, Mitchell, or Lanczos3 algorithms.
*   `create_thumbnail(bytes, maxEdge, format, quality, algorithm, fixExif, backend)`: Fast thumbnail scaling inside a bounding box.
*   `crop_image(bytes, x, y, width, height, format, quality, fixExif)`: Rectangular sub-region extraction.
*   `rotate_image(bytes, rotation, format, quality, fixExif)`: Rotate (90/180/270 degrees) or mirror (horizontal/vertical) images.
*   `compress_image(bytes, format, quality)`: Re-compress image file with given parameters.
*   `read_exif_orientation(bytes)` / `fix_exif_orientation(bytes, format, quality)`: EXIF orientation extraction and physical pixel rewrite.
*   `apply_filter(bytes, filter, format, quality, fixExif)`: Apply single-op adjust adjustments (brightness, contrast, saturation, hue rotate, warmth, fade, vignette, highlights, shadows, structure, preset filters, mood filter presets, swipe looks, LUT PNG filters, or beauty parameters).
*   `add_watermark(baseBytes, overlayBytes, x, y, format, quality)`: Renders watermark overlay.
*   `draw_text_on_image()`, `draw_line_on_image()`, `draw_circle_on_image()`: Direct vector drawing onto compressed image streams.
*   `batch_resize_images(items, algorithm, format, quality, backend)`: Resize multiple image files concurrently in parallel (using `rayon`).
*   `overlay_image(baseBytes, overlayBytes, x, y, blendMode, format, quality)`: Composite overlay with blend modes (Normal, Multiply, Screen, Overlay, Add).
*   `encode_blurhash()`, `decode_blurhash()`: Hash compression conversion.
*   `apply_edit_pipeline(buffer, ops, backend)`: Sequentially executes a list of non-destructive operations (`EditOp`), routing compatible operations to GPU or CPU backends.

### 2. Advanced & Buffer Operations (`api/advanced.rs`)

*   `probe_image(bytes)`: Extract dimensions, format, and EXIF orientation *without* decoding full pixel buffers.
*   `gpu_compute_info()` / `is_gpu_compute_available()`: Detect GPU availability and active device properties.
*   `decode_to_rgba_buffer(bytes, fixExif, maxEdge)`: Decode compressed images directly to raw 32-bit RGBA pixel buffers.
*   `encode_rgba_buffer(buffer, format, quality)`: Encode raw RGBA pixels to compressed formats.
*   `resize_rgba_buffer()`, `crop_rgba_buffer()`, `rotate_rgba_arbitrary()`: Direct pixel operations on raw RGBA arrays.
*   `filter_rgba_buffer(buffer, filter, backend)`: Apply adjustments/filters on raw RGBA buffers.
*   `fit_max_edge_rgba_buffer()`, `encode_rgba_preview_buffer()`: Preview optimizations for raw buffers.
*   `draw_text_rgba_buffer()`, `draw_line_rgba_buffer()`, `draw_circle_rgba_buffer()`: Draw vectors on raw RGBA arrays.
*   `overlay_on_rgba_buffer()`: Blend raw pixel buffers.
*   `decode_progressive_image(bytes, previewMaxEdge, fixExif)`: Progessive decode returning low-res preview buffer + full resolution buffer.
*   `buffer_pool_acquire(minCapacity)` / `buffer_pool_release(buf)` / `buffer_pool_stats()`: Acquire/release vectors from the zero-allocation buffer pool.
*   `processing_backend_name()`, `filter_execution_path_name()`: Diagnostic helper logs.

### 3. Face Analysis & Beauty (`api/face.rs`)

*   `build_skin_mask_from_analysis()`, `build_skin_mask_from_landmarks()`: Generate feathered skin masks at edit resolution.
*   `build_eye_mask_from_landmarks()`, `build_lip_mask_from_landmarks()`, `build_blush_mask_from_landmarks()`, `build_under_eye_mask_from_landmarks()`: Generate regional masks for detailed face segments.
*   `face_analysis_is_valid(analysis)`: Landmark validation thresholds.
*   `apply_skin_smooth_cpu()`, `apply_beauty_cpu()`: Pure CPU fallbacks for beauty filters and skin smoothing.
*   `beauty_params_for_look(preset)`: Look recipes (e.g. Natural, Soft, Glow, Glam, Clear, Peach, Bold).

### 4. Layer Baking (`api/layers.rs`)

*   `bake_layers_on_rgba(buffer, rasterLayers, paintStrokes)`: Bakes multiple sticker/image layers and vector paint strokes (including pen, marker, highlighter, neon brushes, and erasers) onto a base RGBA buffer.

### 5. Live Stream Temporal Smoothing (`api/temporal.rs`)

*   `temporal_smoother_create(alpha)`: Instantiate landmark/mask stabilizer (returns a unique registry ID).
*   `temporal_smoother_smooth(id, raw)`: Apply Exponential Moving Average (EMA) to face coordinates.
*   `temporal_smoother_reset(id)` / `temporal_smoother_destroy(id)`: Reset smoother state or deallocate resources.

### 6. GPU Resident Textures (`api/texture.rs`)

*   `create_gpu_preview_surface(width, height)`: Allocate a GPU-resident preview surface (returns unique handle ID).
*   `destroy_gpu_preview_surface(id)`: Release GPU texture assets.
*   `upload_gpu_preview_surface()`, `readback_gpu_preview_surface()`: Sync CPU buffers to/from GPU memory.
*   `apply_gpu_preview_ops(id, ops, backend)`: Replay edit operations directly on the GPU texture preview cache.
*   `apply_gpu_beauty_pass()`, `apply_gpu_beauty_pipeline()`: Chained WGSL compute shader passes for real-time face cosmetics.
*   `apply_gpu_overlay_blend(id, overlay, opacity, blendMode)`: GPU-side overlay rendering.
*   `is_gpu_texture_preview_available()`: Detect if GPU texture view is supported on the device.

## Requirements

### Host Applications
- **Flutter**: `>= 3.3.0`
- **Dart**: `>= 3.12.0`

### Build Prerequisites
- **Rust Toolchain**: [rustup](https://rustup.rs). Do **not** use Homebrew `rustc` on macOS.
- **Android NDK**: Configured in your `local.properties` (e.g. `ndk.dir=/path/to/ndk`) or the `ANDROID_NDK_HOME` environment variable.
- **Android Targets**: Make sure the target architectures are installed:
  ```bash
  rustup target add aarch64-linux-android armv7-linux-androideabi x86_64-linux-android i686-linux-android
  ```
- **iOS deployment target**: `15.0+` inside your Podfile.

## How to Run, Build & Test

### 1. Bootstrap Workspace
From the monorepo root directory:
```bash
dart pub get && dart run melos bootstrap
```

### 2. Run Crate Unit/Integration Tests
Execute core Rust tests (which cover filters, buffers, and beauty logic):
```bash
cd packages/image_forge/rust
cargo test --features gpu,blurhash
```

### 3. Run Flutter Example App
To run the minimal image processing FFI example (resizes a synthetic buffer and applies filter):
```bash
cd packages/image_forge/example
flutter run -d macos  # or ios / android
```

### 4. Run Benchmarks
Run the native CLI benchmark tool (without Flutter UI overhead):
```bash
cd packages/image_forge/rust
cargo run --release --features gpu --bin rust_image_benchmark -- --synthetic -n 5
```

## Known Issues & Remaining Cleanups

Developers working on or contributing to this package should take note of the following remaining cleanups:

1.  **Unused/Dead Code Warnings**:
    *   Unused imports and helper functions in `src/face/mod.rs` (`warp_mesh::apply_face_warp_rgba`, `recipe_for`, `BeautyRecipe`), `src/layers.rs` (`imageops`), `src/utils.rs` (`ImageEncoder`), `src/compress.rs` (`optimize_bytes`), and `src/filters.rs` (`apply`) need to be pruned or integrated.
2.  **Visibility Warnings**:
    *   `BeautyGpuPipelines` is defined as `pub(crate)` in `src/gpu/beauty_pass.rs`, but is exposed in public signatures of `apply_face_warp_on_cache` and `apply_beauty_on_cache`. It should be refactored to matching public visibility.
3.  **Deferred wgpu surface migration**:
    *   The `GpuEditSurface` / wgpu rendering engine is currently residing in `image_forge/rust`. It is scheduled to be migrated into `pixel_surface` to keep `image_forge` purely headless and isolate-agnostic.
4.  **Android Texture Preview integration**:
    *   Android texture display pipelines are currently under active validation; `useRgbaPreview: true` is recommended on Android for stable fallbacks.
